CSS Reset & Normalize

What is a CSS reset?

Different browsers sometimes have incosistencies in their default styles. Designers use CSS resets to neutralize or change those default styles. This way, the website looks the same across various browsers.

Basically, resets force designers to specifically define style rules for all the different parts of the website.

Common Browser Incosistencies

  • Lists
  • Padding
  • Margins
  • Line Heights
  • Media Elements

Traditional Vs. Modern Resets

Traditional Resets

These resets aim to "aggressively neutralize" browser settings. This creates a blank slate for the designer.

The most common example of a traditional reset is the universal selector:

* {
  margin: 0;
  padding: 0;
}

Drawbacks of Traditional Resets

  • Poor performance caused by heavy use of the universal selector.
  • Overwrites all the browser defaults, even the helpful ones
  • Can disrupt form elements such as <input> or <textarea>

Modern Resets

These modern resets are sometimes called "comprehensive resets". Instead of getting rid of all defaults, they aim to create more refined style baselines that prioritize accessibility and responsive design principles.

Basically, instead of starting with a "blank slate" like the traditional resets, the modern ones improve on what already exists.

Important Components of the Modern Reset

  • Intuitive box model (box-sizing: border-box)
  • Line-heights that make sense and increase readability
  • Standardized typography
  • Logical sizes and defaults for media elements
  • Features that prioritize responsive design, like text-wrap.
